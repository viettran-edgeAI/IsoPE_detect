# Reporting Tools

This directory contains utilities for processing and visualizing raw-PE evaluation
reports generated by the C++ evaluator (`if_quantized_cpp_raw_pe_eval`).

## Purpose

The primary script, `generate_raw_pe_eval_report.py`, parses the text output
produced by the evaluator and produces:

* charts for dataset sample counts (benign vs malware, training vs validation),
  a precision–recall curve, and an ROC curve with log-scaled FPR ticks.
* a markdown summary document containing the measured metrics, model
  parameters, and references to the generated images.

These artifacts are written to a specified `reports/` folder for easy inclusion
in project documentation or analysis workflows.

## Setup & Usage

1. Ensure Python 3.x is available and the repository virtual environment is
   activated (`source .venv/bin/activate`).  Dependencies are limited to
   `matplotlib`; install via:
   ```bash
   pip install matplotlib
   ```

2. Run the reporting tool after a raw‑PE evaluation run has produced the text
   report file (default `development_phase/reports/if_quantized_cpp_raw_pe_eval.txt`):
   ```bash
   python tools/reporting/generate_raw_pe_eval_report.py \
       --report-txt development_phase/reports/if_quantized_cpp_raw_pe_eval.txt \
       --output-dir reports \
       --output-md reports/raw_pe_eval_report.md \
       --quantization-bits 4
   ```

   - `--quantization-bits` is optional and used only for informational text in
     the markdown report.
   - The script will create the output directory if it doesn't exist and write
     PNG charts plus the Markdown file.

3. Open the generated Markdown file or embed the images in other documents
   as needed. The charts and metrics provide a quick visual summary of model
   effectiveness and performance.

That's it — the reporting directory is now ready for use in evaluation
workflows.
